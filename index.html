02-机器学学习大纲

Posted on 2026-01-26

给 2026 年想系统入门并深入机器学习的朋友，下面给出一条“可落地、能就业、可扩展”的 4 阶段路径。

每阶段都标明：

  • ①要掌握的核心技能；
  • ②推荐工具/框架；
  • ③最小实战项目；
  • ④预计周期。你可以按需快进，也可循环迭代。

阶段 1：速成敲门（0-6 周）

目标：跑通一次“数据→模型→预测→上线”全链路，建立信心。

技能

Python 基础（NumPy/Pandas/Seaborn）

经典算法直觉：线性回归、逻辑回归、决策树、K-Means

建模流程：CRISP-DM、交叉验证、指标（ACC/AUC/RMSE）

工具

JupyterLab + scikit-learn 1.5

Gradio/Streamlit 快速做交互 Demo

最小实战

用 Titanic 数据集训练随机森林 → 打包成 Gradio 网页，让同学试玩

资源

免费微课程：Kaggle Learn “Intro to Machine Learning”

图书：《Hands-On Machine Learning》第 2 章～第 6 章

阶段 2：算法深化 + 竞赛刷榜（6-18 周）

目标：能独立做特征工程、调参、集成学习，冲进 Kaggle 前 20%。

技能

模型家族：GBDT（XGBoost/LightGBM）、SVM、聚类、PCA、Ensemble

特征工程：目标编码、频次编码、嵌入、时序滑窗

超参优化：Bayesian Optimization、Optuna

工具

LightGBM 4.x、Optuna、CatBoost

最小实战

“房价预测”或“店铺销量”比赛，单模型突破 Public LB 0.12 → 融合到 0.10

资源

图书：《机器学习实战》（李航）+《百面机器学习》

经验帖：Kaggle Grandmaster 访谈

阶段 3：深度学习与大模型（18-36 周）

目标：理解反向传播/Transformer，能微调大模型，做出端到端 Demo。

技能

CNN/RNN/Transformer 原理；注意力、位置编码、残差

框架：PyTorch 2.x + Hugging Face Ecosystem

任务：文本分类、NER、图文检索、RAG

工具

LoRA/QLoRA、DeepSpeed、LangChain、Llama-Factory

最小实战

用 1 张 24 G 显卡微调 ChatGLM3-6B 做“公司 FAQ 问答”，接入企业微信

资源

图书：《深度学习入门：基于 Python 的理论与实现》

视频：fast.ai 2025 版 Practical Deep Learning

阶段 4：MLOps & 产业落地（36-52 周）

目标：让模型长在生产环境，可灰度、可回滚、可监控。

技能

CI/CD：GitHub Actions + Dockerfile + Kubernetes

数据/模型版本管理：DVC、MLflow、Weights & Biases

监控：Prometheus + Grafana，指标漂移检测

工具

Kubeflow、MLflow 2.x、BentoML、Ollama

最小实战

把阶段 3 的 FAQ 模型封装成 REST/gRPC 服务 → 部署到阿里云 ACK → 配置滚动发布 & 监控

资源

免费课程：Full-Stack Deep Learning 2025

手册：《MLOps 三步法》

加餐模块（贯穿全年）

数学补漏：线性代数（MIT 18.06）、概率（Stat110）、最优化（CS229-notes）

论文复现：每月挑 1 篇“有代码”的顶会论文（ Papers with Code 榜单）

开源贡献：向 Hugging Face、LangChain 提 PR，积累简历亮点

学习节奏建议

70 % 时间写代码跑实验；20 % 读文档/源码；10 % 看数学。

每阶段输出一篇中文技术博客 + GitHub 仓库，形成作品集。

每季度回顾路线图，根据招聘 JD 调整深度（如自动驾驶方向就重点冲多模态+BEV）。

按上面 4 阶段稳扎稳打，一年即可从小白成长为“能独立交付模型+上线”的初级 ML Engineer；第二年再择方向深耕（CV、NLP、强化学习、AI 芯片等），自然水到渠成。祝你学习顺利！

前提检查（1 分钟）

Windows：开始菜单搜索 Anaconda Prompt (miniconda3)

如果返回空，先去官网装 miniconda（体积 100 M 以内，比 Anaconda 轻量）。
安装完关闭终端再重开，让 ~/.bashrc 或 ~/.zshrc 里的 PATH 生效。

因为 Anaconda 在 2024 年末把 repo.anaconda.com 下的默认频道改成了“需先在线接受 ToS（Terms of Service）”模式。

Windows 在 Anaconda Prompt 里执行；macOS/Linux 在终端执行。

执行后会出现 “Terms of Service accepted …” 字样，表示成功。

不更新可能遇到 “Solving environment: failed with initial frozen solve”。

2. 创建干净环境（30 秒）

  • 环境名 ml 可任意换；Python 3.11 是目前（2026-01）与 torch 2.x、scikit-learn 1.5 最稳组合。
  • 不要加 anaconda 元包，否则一次 2 G；我们只装“最小 Python”。

3. 激活环境（Windows/macOS/Linux 通用）

激活后提示符最左边出现 (ml)，表示后续 pip/conda 都装在该环境，不会污染系统。

4.配置国内镜像（2 分钟，一次配置永久生效）

公司内网可用 –trusted-host 或自建 Artifactory；教育网用户可把 tuna 换成 ustc。

装深度学习核心包（CPU 版，1 分钟）

  • 后续有更新 再 pip install –upgrade torch torchvision torchaudio

6. 装科学计算与 Jupyter（1 分钟）

  • 全部用 pip 安装，版本比 conda-forge 新，且与 torch 依赖冲突最少。
  • 如果公司要求 conda 审计，可改 conda install -c conda-forge …，但解决时间可能 3-5 分钟。

7. 装开发辅助工具（30 秒）

black：保存时自动格式化 Python 代码。
nbstripout：提交前自动清除 notebook 输出，避免 Git 冲突。
ipykernel：让 ml 环境出现在 Jupyter 内核列表。

8. 把环境注册成 Jupyter 内核（30 秒）

以后无论系统默认 Python 是什么，Jupyter 里都能选 Python (ml) 内核，保证包隔离。
